# KB010 — New hire & contractor onboarding

**Applies to:** Account creation, laptop prep, standard access for starters and short-term contractors.

## Symptoms
- "New hire starting Monday — create account and assign access."
- Contractor needs a temporary account with limited project access.
- Request to prepare a laptop with the standard software image before a start date.

## L1 resolution steps
1. Confirm start date, manager, department, job title, and required access (groups, shared mailboxes, VPN).
2. Create the user account (UPN, display name, usage location) and assign baseline M365 license.
3. Add the user to approved security/distribution groups; do **not** grant ad-hoc shares without approval.
4. For laptop prep: image or enroll via Intune/Autopilot with the standard build; verify core apps launch.
5. Send welcome instructions (password / SSPR / MFA setup) to the manager or starter per onboarding policy.

## When to escalate
- Non-standard access (admin rights, confidential systems) → manager approval + Identity/Security review.
- Contractor exceeds standard duration or needs elevated access → route to Identity team.
