# KB004 — Software install & licensing

**Applies to:** Approved desktop software, Microsoft 365 / app licenses.

## Symptoms
- Request to install an approved application (e.g., Adobe Acrobat Pro).
- "License expired" or app won't activate.

## L1 resolution steps
1. Confirm the requested software is on the **approved software list**; if not, route to approval workflow.
2. For licensed apps, check license availability before promising a timeline.
3. Deploy via the managed software portal (Company Portal / Intune) where possible.
4. For M365 license issues, verify the user has the correct license assigned in the admin center, then have them sign out/in.
5. Confirm the app launches and activates after install.

## When to escalate
- No licenses available → escalate to licensing/procurement owner.
- Non-standard or paid software needing budget approval → route to the user's manager.
